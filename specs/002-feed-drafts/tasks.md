### T403 Single-save-at-target with fallback in `FeedsController#create` — FR-012, FR-013

- **Files**: `app/controllers/feeds_controller.rb`, `test/controllers/feeds_controller_test.rb`
- **What**: Rewrite `#create` to set the target state from the checkbox, attempt one save, and on enable-failure fall back to a draft save while preserving the enabled-envelope errors. Sketch:
  ```ruby
  def create
    @feed = feeds_scope.build(create_feed_params)
    authorize @feed

    @feed.preview_token = params[:preview_token]
    @feed.state = params[:enable_feed] == "1" ? :enabled : :draft

    if save_with_optional_promotion(@feed)
      cleanup_feed_identification(@feed.url) if @feed.url
      redirect_to feed_path(@feed), notice: success_message_for(@feed)
    else
      render :new, status: :unprocessable_entity
    end
  end

  private

  # Returns true if @feed is persisted in some valid form (enabled, or draft as fallback).
  # Populates @feed.errors with enabled-envelope errors when promotion failed.
  def save_with_optional_promotion(feed)
    return true if feed.save

    return false unless feed.state == "enabled"

    enabled_errors = feed.errors.deep_dup
    feed.state = :draft
    if feed.save
      enabled_errors.each { |error| feed.errors.import(error) }
      flash.now[:alert] = "Saved as draft. Fix the issues below to enable."
      true
    else
      false
    end
  end
  ```
  Wait — if `save_with_optional_promotion` returns true on the fallback path, we'd `redirect_to feed_path(@feed)` and lose the form errors. We need a different return shape: success means "redirect"; fallback success means "re-render with errors AND data persisted." Restructure:
  ```ruby
  def create
    @feed = feeds_scope.build(create_feed_params)
    authorize @feed
    @feed.preview_token = params[:preview_token]
    @feed.state = params[:enable_feed] == "1" ? :enabled : :draft

    case attempt_save(@feed)
    when :enabled, :draft_saved
      cleanup_feed_identification(@feed.url) if @feed.url
      redirect_to feed_path(@feed), notice: success_message_for(@feed)
    when :draft_fallback
      render :new, status: :unprocessable_entity  # data persisted, errors shown, flash set
    when :failed
      render :new, status: :unprocessable_entity
    end
  end

  # Outcomes:
  #   :enabled         — saved at target enabled state
  #   :draft_saved     — saved at target draft state (user didn't request enable)
  #   :draft_fallback  — user requested enable, validation failed, persisted as draft with errors
  #   :failed          — couldn't save even as draft
  def attempt_save(feed)
    attempting_enable = feed.enabled?
    if feed.save
      attempting_enable ? :enabled : :draft_saved
    elsif attempting_enable
      enabled_errors = feed.errors.map { |error| [error.attribute, error.message] }
      feed.state = :draft
      if feed.save
        enabled_errors.each { |attribute, message| feed.errors.add(attribute, message) }
        flash.now[:alert] = "Saved as draft. Fix the issues below to enable."
        :draft_fallback
      else
        :failed
      end
    else
      :failed
    end
  end
  ```
  Note: capturing `(attribute, message)` tuples and re-adding via `errors.add` is the most Rails-version-portable way to preserve the enabled-envelope errors across the fallback save. If the project's Rails supports `ActiveModel::Errors#copy!` or `#import` cleanly, those are equivalent.
- **Test additions**:
  - `test "#create should save and enable when checkbox checked and all fields valid"` — POST with `enable_feed=1` and full config; expect redirect to feed show, `feed.enabled?`.
  - `test "#create should save as draft when checkbox unchecked"` — POST without `enable_feed`; expect redirect, `feed.draft?`.
  - `test "#create should fall back to draft when enable-validation fails but draft-validation passes"` — POST with `enable_feed=1` but missing `target_group`; expect `:unprocessable_entity`, `Feed.last.draft?`, response body includes the target_group error.

  - `test "#create should fail without persisting when even draft validation fails"` — POST with no `feed_profile_key`; expect `:unprocessable_entity`, no Feed row created.
- **Commit message**: `FeedsController#create: single-save-at-target with draft fallback on enable failure`

### T404 Single-save-at-target with fallback in `FeedsController#update` — FR-012, FR-013, FR-014

- **Files**: `app/controllers/feeds_controller.rb`, `test/controllers/feeds_controller_test.rb`
- **What**: Apply the same pattern in `#update`. The target state derivation differs slightly: for existing records the "fallback" state is the feed's prior persisted state (typically `:disabled`, since drafts that fail to enable stay drafts; disabled feeds that fail to re-enable stay disabled).
  ```ruby
  def update
    @feed = load_feed
    authorize @feed

    @feed.preview_token = params[:preview_token]
    @feed.assign_attributes(update_feed_params)

    if params[:enable_feed] == "1"
      @feed.state = :enabled
      fallback = @feed.state_was.to_sym
    else
      @feed.state = (@feed.enabled? ? :disabled : @feed.state)
      fallback = nil  # no fallback needed; we're not attempting promotion
    end

    case attempt_update_save(@feed, fallback)
    when :saved
      reset_schedule_if_interval_changed
      cleanup_feed_identification(@feed.url) if @feed.url
      redirect_to feed_path(@feed), notice: update_message_for(@feed)
    when :fallback_saved, :failed
      render :edit, status: :unprocessable_entity
    end
  end
  ```
  with `attempt_update_save(feed, fallback)` analogous to `attempt_save` but using the passed-in fallback state. Reuse helper logic between `#create` and `#update` if it's clean to extract.
